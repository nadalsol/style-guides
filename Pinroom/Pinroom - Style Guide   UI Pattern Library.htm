Grid units (based on 960.gs)

  • Ventanas modales (dialogs): grid 12 columnas.
  • Páginas "no masonrizadas": grid 16 columnas.
  • Páginas "masonrizadas": no usaremos grid ya que cada brick se posiciona de manera absoluta (top y left) a partir de un contenedor. Aunque se podría usar un grid tiene sentido no usarlo ya que queremos mostrar los bricks en pantalla en el mismo orden que cargan en el código. Esto nos permite usar Lazy Load.

Media Object

El Media Object lo usaremos para la típica estructura de imagen + descripción. Se compone por una estructura tipo Media Imagen (mediaImg) + Media Body (mediaBd). Posibles configuraciones:

  • Imagen a la izquierda con enlace, con contenido descriptivo a la derecha.
  • Imagen a la izquierda sin enlace, con contenido descriptivo a la derecha.
  • Imagen a la derecha con enlace, con contenido descriptivo a la izquierda.
  • Imagen a la derecha sin enlace, con contenido descriptivo a la izquierda.

El Media Object se puede utilizar con todo tipo de media u objetos de bloque, no sólo con imágenes: párrafos, divs, videos, etc… Sólo hay que aplicar las mismas classes que en el caso de las imágenes.

  • Dialog Type 1: Title + Close icon.
  • Dialog Type 2: Title + Close icon (with Subtitle).
  • Dialog Type 3: Title + Close icon (with Label).

Forms

Los formularios normalmente aparecen dentro de dialogs, aunque si es necesario pueden ir fuera de estos. Posibles configuraciones, en función de los distintos layouts de formulario:

  • Form Type 1: Single column with Fixed width inputs (contiguous notification messages)
  • Form Type 2: Single column with Fixed width inputs & Visible Labels
  • Form Type 3: Multiple columns with 100% width inputs (notification messages beneath)

Form Type 1: Single column with Fixed width inputs (contiguous notification messages)

Los ficheros editables de los sprites (en formato PSD de Photoshop) están en Dropbox (Dropbox > Pinroom (1) > Sprites_PSD), por si hay que hacer cambios o crear versiones nuevas de los mismos.

Disponemos de distintos tipos de sprites, en función de su naturaleza:

  • General Sprite
  • Privacy Icons Sprite
  • Star Rating Sprite
  • Badges Sprite

Para el General Sprite usaremos el tag img con una imagen transparente (inv.gif), junto con la class "s" seguida de la class correspondiente al sprite en concreto que queramos usar, p.e. "typeArticle".

→ Mostrar código 

<img src="img/inv.gif" width="41" height="41" class="s typeArticle" alt="Artículo" />

El Privacy Icons Sprite los usaremos en botones junto con un tag span con la class "sPrivacy" seguida de la class correspondiente al icono del candado que queramos usar ("private" o "public").

    <span class="sPrivacy private">privado</span>
</a>

Star Rating Sprite

El Star Rating Sprite forma parte del plugin jQuery Simple Half-Star Rating Plugin. Usa un tag div con class "star" seguida de otra class "star-left" o "star-right", seguida de la class correspondiente al estado (on / off / hover). El siguiente código es a modo de ejemplo, ya que la asignación de classes, data attributes y demás es gestionado automáticamente por el plugin.

→ Mostrar código 

<div class="rating">
    <div class="star star-left on"><a href="#" title="1/5" style="width:100%;">1</a></div>
    <div class="star star-right on"><a href="#" title="1/5" style="width:100%;">1</a></div>
    <div class="star star-left on"><a href="#" title="2/5" style="width:100%;">2</a></div>
    <div class="star star-right on"><a href="#" title="2/5" style="width:100%;">2</a></div>
    <div class="star star-left on"><a href="#" title="3/5" style="width:100%;">3</a></div>
    <div class="star star-right"><a href="#" title="3/5" style="width:100%;">3</a></div>
    <div class="star star-left"><a href="#" title="4/5" style="width:100%;">4</a></div>
    <div class="star star-right"><a href="#" title="4/5" style="width:100%;">4</a></div>
    <div class="star star-left"><a href="#" title="5/5" style="width:100%;">5</a></div>
    <div class="star star-right"><a href="#" title="5/5" style="width:100%;">5</a></div>
